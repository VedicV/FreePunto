# Продуктовий план

## Як я розумію продукт

Потрібно зробити macOS-застосунок у рядку меню, який не виправляє текст автоматично, а виконує тільки ручні команди користувача.

Головна команда: взяти виділений текст, а якщо виділення немає - останнє слово перед курсором, і перетворити його одним зі способів:

- змінити текст між англійською, російською та українською клавіатурними розкладками;
- після перетворення перемкнути активну системну розкладку macOS на мову результату;
- змінити регістр;
- виконати фонетичну транслітерацію між кирилицею та латиницею;
- тимчасово призупинити застосунок.

Принцип першої версії: тільки потрібні функції, без автокорекції, словників, вгадування та фонового надсилання тексту.

## Базова поведінка

### Область дії

1. Якщо є виділений текст - перетворюється виділення.
2. Якщо виділення немає - перетворюється останнє слово перед курсором.
3. Застосунок не запускає автокорекцію сам.
4. Застосунок не зберігає історію введення.
5. Застосунок не надсилає текст у мережу.

### Основна клавіша

За замовчуванням основна дія викликається одиночним натисканням `Control`.

Також можна дозволити звичайні комбінації клавіш, якщо користувач захоче замінити одиночну клавішу.

Для macOS треба окремо перевірити:

- одиночне натискання `Control`;
- лівий і правий `Control`;
- конфлікт із системними комбінаціями клавіш.

Якщо одиночний `Control` в якомусь середовищі заважатиме системним комбінаціям, користувач зможе вибрати явну комбінацію.

## Перемикання мов

Потрібні два режими.

### Послідовний режим

Повторні натискання перебирають варіанти по колу.

Якщо початковий текст англійський:

```text
Англійська -> російська -> українська -> англійська
```

Якщо початковий текст російський:

```text
Російська -> англійська -> українська -> російська
```

Якщо початковий текст український:

```text
Українська -> англійська -> російська -> українська
```

Це також замінює окремий власний відкат: якщо потрібно повернути початковий варіант, користувач може натискати основну клавішу далі до повернення по колу. Де можливо, системний `Cmd+Z` цільового застосунку теж має працювати нативно.

### Режим фіксованої цілі

Більш передбачуваний режим:

- російський і український текст перетворюються на англійський;
- англійський текст перетворюється на вибрану мову: російську або українську.

Після перетворення застосунок перемикає системну розкладку macOS на мову результату.

## Функції першої версії

- macOS-застосунок у рядку меню.
- Глобальна гаряча клавіша основного перетворення.
- Одиночний `Control` за замовчуванням.
- Налаштування гарячих клавіш.
- Перетворення виділеного тексту.
- Перетворення останнього слова, якщо виділення немає.
- Перекладка тексту між англійською, російською та українською розкладками за таблицею фізичних клавіш.
- Перемикання системної розкладки macOS на мову результату.
- Послідовний режим.
- Режим фіксованої цілі.
- Гаряча клавіша для зміни регістру.
- Гаряча клавіша для транслітерації.
- Тимчасова пауза.
- Мінімальне меню.
- Мінімальні налаштування.
- Іконка в рядку меню, що показує наступну мову або паузу.

## Регістр

Потрібна окрема команда для перетворення регістру:

- нижній регістр - усі літери маленькі;
- регістр речення - перша літера речення велика, інші звичайні;
- великі перші літери слів.

Окремий режим "усі літери великі" зараз не потрібен.

Команда має зберігати:

- пробіли;
- переноси рядків;
- пунктуацію;
- межі виділеного тексту.

## Транслітерація

Потрібна окрема команда для звичної фонетичної транслітерації:

- російський текст кирилицею -> латиниця;
- український текст кирилицею -> латиниця;
- латиниця -> російський текст кирилицею;
- латиниця -> український текст кирилицею.

Для напряму `латиниця -> кирилиця` потрібен вибір цільової мови: російська або українська.

Важливо не змішувати транслітерацію з перемиканням розкладки:

- перемикання розкладки: `ghbdtn` -> `привет`;
- транслітерація: `привет` -> `privet`.

## Мінімальне меню

Меню в рядку меню має містити тільки потрібне:

- увімкнено / пауза;
- виконати основне перетворення;
- змінити регістр;
- транслітерувати;
- режим перемикання: послідовний або фіксована ціль;
- цільова мова для фіксованого режиму: російська або українська;
- цільова мова для транслітерації з латиниці: російська або українська;
- налаштування гарячих клавіш;
- запускати при вході в систему;
- вихід.

## Іконка

Іконка має показувати:

- `EN`, `RU`, `UA` - наступну мову для основного перетворення;
- стан паузи;
- за можливості маленьку ознаку режиму: послідовний або фіксований.

Якщо надійно визначати мову поточного фрагмента занадто складно, перша версія може показувати поточне налаштування режиму й очікувану наступну ціль.

## Що не входить до першої версії

- Автокорекція.
- Словники.
- Визначення мови через ML, AI або n-грами.
- Підказки та спливні виправлення.
- Винятки для застосунків.
- Власна історія дій.
- Власний механізм відкату.
- Синхронізація налаштувань.
- Підтримка Windows і Linux.
- Розширення та плагіни.
- Історія введеного тексту.

## Етапи

### Етап 0 - перевірка можливостей macOS

- Перевірити одиночний `Control` як глобальний тригер.
- Перевірити читання виділеного тексту через Accessibility.
- Перевірити запасний спосіб через буфер обміну з обов'язковим відновленням попереднього вмісту.
- Перевірити заміну останнього слова без виділення.
- Перевірити перемикання системної розкладки macOS через Text Input Source APIs.
- Перевірити, як операції потрапляють у системний стек відкату цільового застосунку.

### Етап 1 - перша версія для виділеного тексту

- Застосунок у рядку меню.
- Налаштування гарячих клавіш.
- Перетворення виділеного тексту між EN/RU/UA.
- Послідовний режим і режим фіксованої цілі.
- Перемикання системної розкладки після перетворення.
- Пауза.
- Статусна іконка.

### Етап 2 - останнє слово

- Визначення останнього слова перед курсором.
- Заміна останнього слова без пошкодження навколишнього тексту.
- Повернення по колу через повторне перетворення.
- Тести в TextEdit, Notes, Safari/Chrome, Messages/Telegram, редакторі коду.

### Етап 3 - регістр і транслітерація

- Нижній регістр.
- Регістр речення.
- Великі перші літери слів.
- Фонетична транслітерація RU/UA <-> EN.
- Налаштування напряму транслітерації.
- Тести на змішаному тексті, пунктуації та багаторядкових виділеннях.

### Етап 4 - полірування та реліз

- Первинний екран налаштування дозволів macOS.
- Code signing і notarization.
- Мінімальна діагностика без збереження тексту.
- Пакування релізної збірки.
