# Встановлення та використання FreePunto

FreePunto - локальний macOS-застосунок у рядку меню для ручного перетворення тексту між англійською, російською та українською розкладками. Застосунок не робить автокорекцію: він працює тільки після вашої команди.

## Вимоги

- macOS 14 або новіше.
- Xcode або Command Line Tools зі Swift toolchain.
- Додані в macOS розкладки, які ви хочете використовувати: English, Russian, Ukrainian.

## Збирання застосунку

З кореня проєкту виконайте:

```bash
./scripts/build_app.sh
```

Скрипт збирає release-версію Swift Package і створює локальний bundle:

```text
dist/FreePunto.app
```

Для розробки можна також запускати застосунок без `.app` bundle:

```bash
swift run FreePunto
```

Але для перевірки поведінки як звичайного застосунку зручніше використовувати `dist/FreePunto.app`.

## Запуск

Після збирання відкрийте застосунок:

```bash
open dist/FreePunto.app
```

FreePunto не відкриває звичайне вікно. Після запуску він з'являється в рядку меню macOS як маленька вузька іконка-клавіша:

- у нижньому лівому куті показано символ основної клавіші `⌃`;
- у верхньому правому куті показано наступну мову `EN`, `RU` або `UA`;
- якщо застосунок на паузі, у верхньому правому куті показано значок паузи з двох вертикальних смуг.

При наведенні курсора macOS показує текстову підказку на кшталт `FreePunto: RU` або `FreePunto: PAUSE`.

## Збирання DMG

Щоб зібрати стандартний інсталяційний `.dmg`, виконайте:

```bash
./scripts/build_dmg.sh
```

Підсумковий файл:

```text
dist/FreePunto-0.1.1.dmg
```

Усередині DMG лежить `FreePunto.app` і ярлик `Applications`. Відкрийте DMG і перетягніть `FreePunto.app` до `Applications`.

## Встановлення в Applications

Для постійного використання можна перенести зібраний застосунок у користувацьку папку Applications:

```bash
mkdir -p ~/Applications
cp -R dist/FreePunto.app ~/Applications/FreePunto.app
open ~/Applications/FreePunto.app
```

Якщо ви перезібрали застосунок, закрийте FreePunto, замініть старий `~/Applications/FreePunto.app` новим і запустіть знову.

## Дозволи macOS

Для глобальних гарячих клавіш, читання виділеного тексту і заміни тексту macOS може вимагати дозволи:

- Accessibility.
- Input Monitoring.

Відкрийте `System Settings -> Privacy & Security` і додайте FreePunto до:

- `Accessibility`;
- `Input Monitoring`, якщо гарячі клавіші або одиночний `Control` не спрацьовують.

Після зміни дозволів перезапустіть FreePunto. У меню застосунку також є пункт `Дозволи...`, який показує поточну підказку щодо дозволів.

## Unsigned / not notarized

Локальна збірка з `./scripts/build_app.sh` не підписана і не нотаризована. Це нормально для розробки, але macOS може заблокувати перший запуск.

Якщо macOS повідомляє, що застосунок не можна відкрити, спробуйте:

1. Відкрити `System Settings -> Privacy & Security` і дозволити запуск заблокованого застосунку.
2. Запустити FreePunto ще раз.

Якщо застосунок був завантажений або перенесений між машинами і macOS тримає quarantine-мітку, для локальної розробки можна зняти її командою:

```bash
xattr -dr com.apple.quarantine dist/FreePunto.app
```

Використовуйте цю команду тільки для збірки, якій довіряєте.

## Меню в рядку меню

Натисніть на іконку FreePunto в рядку меню. За замовчуванням меню українською, але мову можна змінити в пункті `Мова меню`.

Основні пункти меню:

- `Пауза` / `Продовжити` - тимчасово призупинити або увімкнути застосунок.
- `Змінити розкладку (Control)` - перетворити розкладку виділеного тексту або останнього слова.
- `Змінити регістр (Control+Option+C)` - змінити регістр за вибраним режимом.
- `Транслітерація (Control+Option+T)` - виконати транслітерацію.
- `Режим перемикання` - вибрати режим по колу або фіксовану ціль.
- `Ціль розкладки` - вибрати фіксовану ціль `RU` або `UA`.
- `Ціль транслітерації` - вибрати цільову кириличну мову `RU` або `UA` для транслітерації з латиниці.
- `Регістр` - вибрати режим регістру.
- `Мова меню` - вибрати український, російський або англійський інтерфейс.
- `Гарячі клавіші та налаштування...` - відкрити налаштування.
- `Запускати при вході` / `Не запускати при вході` - увімкнути або вимкнути запуск при вході в систему.
- `Дозволи...` - показати підказку щодо системних дозволів.
- `Вийти з FreePunto` - закрити застосунок.

## Гарячі клавіші

За замовчуванням:

- Основне перетворення розкладки: одиночний `Control`.
- Зміна регістру: `Control+Option+C`.
- Транслітерація: `Control+Option+T`.
- Пауза: `Control+Option+P`.

Головна клавіша зміни розкладки - саме `Control`. У налаштуваннях вона показана окремим рядком `Зміна розкладки`. Там можна:

- побачити поточне значення, за замовчуванням `Control`;
- натиснути `Записати` і задати іншу комбінацію;
- натиснути `Повернути Control`, щоб знову використовувати одиночний `Control`.

Для інших дій також можна натиснути `Записати` поруч із потрібним рядком і задати свою комбінацію.

### Одиночний Control

Одиночний `Control` - основний тригер за замовчуванням. Натисніть і відпустіть лівий або правий `Control` без інших клавіш, щоб виконати `Змінити розкладку`.

Якщо такий тригер заважає вашим звичним комбінаціям або не працює в конкретному середовищі:

1. Відкрийте `Гарячі клавіші та налаштування...`.
2. У рядку `Зміна розкладки` натисніть `Записати`.
3. Натисніть нову комбінацію.

## Як користуватися перетворенням

Звичайний сценарій:

1. Виділіть текст у будь-якому застосунку.
2. Натисніть одиночний `Control` або вибрану гарячу клавішу.
3. FreePunto замінить виділення перетвореним текстом.
4. Якщо можливо, FreePunto перемкне системну розкладку macOS на мову результату.

Якщо текст не виділений, FreePunto намагається вибрати й перетворити останнє слово перед курсором.

Перетворення працює локально. FreePunto не надсилає текст у мережу і не зберігає історію введення.

## Режими розкладки

### По колу

Режим `По колу` перебирає мови по колу. Повторна дія на тому самому фрагменті дає наступний варіант.

Приклади циклів:

- англійська -> російська -> українська -> англійська;
- російська -> англійська -> українська -> російська;
- українська -> англійська -> російська -> українська.

Цей режим зручний, коли потрібно швидко виправляти текст і іноді повертатися до попереднього варіанта повторним натисканням.

### Фіксована ціль

Режим `Фіксована ціль` використовує фіксовану ціль:

- російський і український текст перетворюється на англійський;
- англійський текст перетворюється на `RU` або `UA`, залежно від `Ціль розкладки`.

У рядку меню фіксований режим позначається маленькою крапкою на мовній іконці.

## Регістр

Дія `Змінити регістр` застосовує режим із меню `Регістр`:

- `Нижній регістр` - усі літери маленькі.
- `Регістр речення` - перша літера речення велика.
- `Перші літери слів` - перші літери слів великі.
- `Виправити Caps Lock` - привести текст, набраний ніби з Caps Lock, до звичайного нижнього регістру.

Режим можна вибрати в меню `Регістр` або у вікні `Гарячі клавіші та налаштування...`.

## Транслітерація

Дія `Транслітерація` перетворює текст між кирилицею та латиницею:

- російська кирилиця -> латиниця;
- українська кирилиця -> латиниця;
- латиниця -> російська або українська кирилиця.

Для напряму з латиниці виберіть ціль у меню `Ціль транслітерації`: `RU` або `UA`.

Транслітерація використовує звичну фонетичну відповідність, а не офіційний паспортний стандарт.

## Пауза

Пауза тимчасово вимикає перетворення. Її можна увімкнути через:

- пункт меню `Пауза`;
- гарячу клавішу паузи;
- чекбокс `Увімкнено` в `Гарячі клавіші та налаштування...`.

Коли застосунок на паузі, у рядку меню показано значок паузи з двох вертикальних смуг. Гаряча клавіша паузи продовжує працювати, щоб можна було швидко увімкнути FreePunto назад.

## Запуск при вході в систему

Щоб запускати FreePunto автоматично при вході в macOS:

1. Відкрийте меню FreePunto в рядку меню.
2. Натисніть `Запускати при вході`.

Вимкнення робиться тим самим пунктом, який після ввімкнення називається `Не запускати при вході`.

Якщо macOS не дозволяє ввімкнути автозапуск, FreePunto покаже помилку. Перевірте системні налаштування входу в обліковий запис і спробуйте знову.

## Troubleshooting

### FreePunto не з'являється в Dock

Це очікувано. FreePunto - menu bar застосунок. Шукайте маленьку іконку `EN`, `RU`, `UA` або значок паузи в рядку меню macOS.

### Гарячі клавіші не працюють

Перевірте:

- FreePunto запущений і не перебуває в стані паузи.
- FreePunto доданий до `Accessibility`.
- FreePunto доданий до `Input Monitoring`.
- Після надання дозволів застосунок було перезапущено.
- Інша програма не перехоплює ту саму гарячу клавішу.

Якщо одиночний `Control` не підходить, задайте явну комбінацію в `Гарячі клавіші та налаштування...`.

### Текст не замінюється

Перевірте:

- чи є виділений текст;
- чи знаходиться курсор у звичайному текстовому полі;
- чи не захищене поле системними обмеженнями, наприклад поле пароля;
- чи дозволений FreePunto в `Accessibility`.

Деякі застосунки можуть не віддавати виділений текст через Accessibility. У такому разі FreePunto намагається працювати через виділення останнього слова і буфер обміну, але це теж може бути обмежено конкретним застосунком.

### Після перетворення не перемикається розкладка

Додайте потрібну розкладку в `System Settings -> Keyboard -> Text Input -> Input Sources`. FreePunto може перемкнутися тільки на доступний у системі input source.

### З'являється помилка `Гарячі клавіші недоступні`

Зазвичай це означає, що macOS не дала створити глобальний event tap. Надайте дозволи `Accessibility` і `Input Monitoring`, потім повністю закрийте й знову запустіть FreePunto.

### macOS не відкриває застосунок

Локальна збірка не підписана і не нотаризована. Дозвольте запуск у `System Settings -> Privacy & Security` або, для довіреної локальної збірки, зніміть quarantine-мітку:

```bash
xattr -dr com.apple.quarantine dist/FreePunto.app
```

### Запуск при вході не вмикається

Перевірте, що ви запускаєте саме `.app` bundle з `dist/FreePunto.app`, а не тільки `swift run FreePunto`. Для перевірки автозапуску поведінка `.app` ближча до реального використання.
