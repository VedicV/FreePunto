# Архитектура

## Направление

Первая версия - нативное macOS-приложение в строке меню на Swift, SwiftUI и AppKit.

Главная задача архитектуры - надежные ручные преобразования текста. Автокоррекции, словарей и фонового анализа языка нет.

## Основные модули

```text
Приложение в строке меню
  -> Контроллер горячих клавиш
  -> Определение цели текста
  -> Движок преобразований
      -> Преобразование раскладки
      -> Преобразование регистра
      -> Транслитерация
      -> Выбор следующего языка
  -> Замена текста
  -> Переключение системной раскладки
  -> Хранилище настроек
  -> Статусная иконка
  -> Диагностика разрешений
```

## Поток действия

```text
Пользователь нажимает горячую клавишу
  -> контроллер определяет команду
  -> приложение берет выделенный текст или последнее слово
  -> движок строит замену
  -> приложение заменяет текст
  -> приложение переключает системную раскладку на язык результата
  -> статусная иконка обновляет состояние
```

Фонового потока автокоррекции нет.

## macOS API, которые нужно проверить

### Горячие клавиши

Нужно проверить:

- глобальные сочетания клавиш;
- одиночное нажатие `Control`;
- отличие одиночного нажатия от сочетания с другими клавишами;
- левый и правый `Control`;
- конфликт с системными сочетаниями клавиш.

Риск: одиночный `Control` может конфликтовать с системными или пользовательскими сочетаниями клавиш. Поэтому выбор горячей клавиши должен быть настройкой с самого начала.

### Чтение выделенного текста

Предпочтительный порядок:

1. Accessibility API для выбранного текста, где это возможно.
2. Запасной способ через буфер обмена: скопировать выделение, прочитать буфер обмена, восстановить прежнее содержимое.

Запасной способ через буфер обмена должен восстанавливать прежнее содержимое и не сохранять его.

### Последнее слово

Возможные стратегии:

1. Короткий in-memory буфер недавно введенных символов.
2. Выделение предыдущего слова системной навигацией, чтение, замена и восстановление курсора.
3. Accessibility text range APIs, если конкретное приложение их поддерживает.

Первую версию лучше начать с выделенного текста, затем добавить последнее слово после проверки надежной замены.

### Замена текста

Предпочтительный порядок:

1. Accessibility API replacement, где это возможно.
2. Замена через вставку из буфера обмена с восстановлением прежнего содержимого.
3. Синтетические key events только если другого надежного пути нет.

Замена должна по возможности попадать в нативный стек отмены целевого приложения. Отдельный собственный откат не реализуем.

### Переключение системной раскладки

После успешного преобразования нужно переключить активную раскладку macOS на язык результата:

- результат английский -> английская раскладка;
- результат русский -> русская раскладка;
- результат украинский -> украинская раскладка.

Для этого нужно проверить Text Input Source APIs:

- получение доступных источников ввода;
- выбор источника ввода по языку;
- поведение, если нужная раскладка не добавлена в настройках macOS;
- запасной сценарий: показать пользователю, что нужную раскладку надо добавить.

## Движок преобразований

### Преобразование раскладки

Преобразует текст по физической клавиатурной карте:

- EN <-> RU;
- EN <-> UA;
- RU <-> UA через общую карту физических клавиш.

Пример:

```text
ghbdtn -> привет
руддщ -> hello
```

Это не транслитерация.

### Выбор следующего языка

Модуль определяет язык результата.

Последовательный режим:

```text
Английский -> русский -> украинский -> английский
Русский    -> английский -> украинский -> русский
Украинский -> английский -> русский -> украинский
```

Режим фиксированной цели:

```text
Русский/украинский -> английский
Английский -> выбранный кириллический язык
```

Для повторных нажатий по одному и тому же фрагменту желательно хранить короткий контекст последнего преобразования, чтобы цикл шел предсказуемо и мог вернуть исходник.

### Преобразование регистра

Поддерживаем:

- нижний регистр;
- регистр предложения;
- заглавные первые буквы слов;
- исправление текста, случайно набранного с Caps Lock, в обычный нижний регистр.

Отдельный режим "все буквы заглавные" не нужен.

### Транслитерация

Фонетическая транслитерация:

- русский -> латиница;
- украинский -> латиница;
- латиница -> русский;
- латиница -> украинский.

Транслитерация и преобразование раскладки - разные операции и разные таблицы.

```text

целевой_язык_транслитерации: русский | украинский
режим_статусной_иконки
```

## Минимальный интерфейс

Используем элемент в строке меню и компактное окно настроек.

Окно настроек должно включать только:

- горячие клавиши;
- режим переключения;
- целевой язык;
- целевой язык транслитерации;
- запуск при входе;
- статус разрешений macOS.

## Граница приватности

Так как автокоррекции нет, приложению не нужно постоянно анализировать весь ввод.

Для последнего слова, если понадобится буфер:

- хранить только недавно введенные символы;
- хранить только в памяти;
- очищать при смене фокуса;
- очищать при паузе;
- очищать при переходе в сон и блокировке экрана.

Содержимое текста не пишется на диск и не отправляется в сеть.

## Тесты

### Модульные тесты

- Карты EN/RU/UA.
- Обратимость преобразований.
- Последовательный режим.
- Режим фиксированной цели.
- Переключение целевой раскладки.
- Нижний регистр.
- Регистр предложения.
- Заглавные первые буквы слов.
- Фонетическая транслитерация.
- Сохранение пунктуации и пробелов.

### Ручные интеграционные тесты

- TextEdit.
- Notes.
- Safari и Chrome.
- Адресная строка браузера.
- Messages или Telegram.
- VS Code или другой редактор.
- Поля паролей: проверить, что приложение не отправляет ничего в сеть и ведет себя настолько хорошо, насколько позволяют macOS API.
