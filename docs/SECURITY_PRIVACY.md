# Безпека та приватність

## Позиція щодо безпеки

Застосунок не робить автокорекцію і не має поводитися як кейлогер. Він бере текст тільки для виконання явної команди користувача: перетворити виділення або останнє слово.

Головне правило: текст не йде в мережу і не зберігається як історія.

## Правила

- Немає телеметрії.
- Немає хмарної обробки.
- Немає надсилання тексту в мережу.
- Немає історії натискань клавіш.
- Немає історії введеного тексту.
- Немає історії буфера обміну.
- Немає автокорекції.
- Немає словників і навчання на користувацькому тексті в першій версії.
- Немає збереження початкового або перетвореного тексту на диск.

## Робота з текстом

Звичайний сценарій:

1. Користувач натискає гарячу клавішу.
2. Застосунок читає виділений текст або останнє слово.
3. Застосунок перетворює текст локально.
4. Застосунок замінює текст.
5. Застосунок забуває текст.

Якщо для останнього слова знадобиться короткий буфер, він має бути:

- тільки в пам'яті;
- обмежений за довжиною;
- очищатися при зміні фокуса;
- очищатися при паузі;
- очищатися при переході в сон і блокуванні екрана.

## Запасний спосіб через буфер обміну

Якщо буфер обміну використовується для читання або заміни тексту:

- зберегти попередній вміст буфера обміну;
- відновити попередній вміст після операції;
- не логувати буфер обміну;
- не зберігати буфер обміну;
- не надсилати буфер обміну в мережу;
- акуратно обробити помилку, якщо відновлення буфера обміну не вдалося.

## Поля паролів і захищене введення

У першій версії не робимо користувацькі винятки для застосунків і не вводимо спеціальне правило "не працювати в полях паролів".

Водночас застосунок не має намагатися обходити системні обмеження macOS. Якщо macOS або конкретний застосунок не дає прочитати чи замінити текст у захищеному полі, команда має просто не виконатися.

Ключова вимога: навіть якщо користувач перетворює текст у полі пароля, початковий і перетворений текст залишаються локально, не пишуться в історію і не надсилаються в мережу.

## Дозволи macOS

Ймовірно знадобляться:

- Accessibility - для читання/заміни виділеного тексту та керування текстовими полями;
- Input Monitoring - якщо одиночний `Control` або останнє слово вимагатимуть відстеження введення.

Застосунок має пояснювати, навіщо потрібен кожен дозвіл. Якщо дозволу немає, застосунок має працювати в обмеженому режимі або показувати зрозумілу причину, чому команда недоступна.

## Діагностика

Діагностика не має містити користувацький текст.

Можна зберігати:

- стан дозволів;
- тип команди;
- код успіху або помилки;
- час виконання;
- поточний режим;
- мову результату.

Не можна зберігати:

- початковий текст;
- перетворений текст;
- вміст буфера обміну;
- історію введення;
- повні заголовки активних вікон.

## Мережа

Для основної роботи застосунку мережа не потрібна.

У першій версії краще взагалі не додавати мережеві функції. Оновлення можна робити вручну через підписані релізи.

## Релізна безпека

- Підписати застосунок.
- Нотаризувати застосунок.
- Зафіксувати залежності.
- Не додавати автооновлення, поки немає підписаного й перевірюваного механізму оновлень.
